# 토큰 ergonomics 테스트 프롬프트 — 결제/플랜 비교 페이지

아래 두 프롬프트를 **각각 별개의 새 대화**에서 실행한다.
프롬프트 내용은 import 경로와 출력 파일 경로만 다르고 UI 스펙은 동일하다.

이 테스트의 목적: `surface.brand.*` + `fill.brand` + `border.brand.*`를 플랜 카드 강조에 집중 사용하고, 결제 내역 테이블의 상태 뱃지(완료/실패/처리중)로 positive/critical/warning 계열을 반복 검증.

## 페이지 스펙: 결제 및 구독 관리

### 레이아웃
왼쪽 사이드바 (고정 240px) + 오른쪽 메인 콘텐츠 (스크롤), 전체 100vh.
사이드바는 대시보드와 동일한 어두운 solid 계열 스타일로 구성.

### 사이드바
- 배경: 어두운 solid 계열
- 상단: 로고 아이콘 (LayoutDashboard) + "Mildang" 텍스트
- 네비 메뉴: 결제 내역 (CreditCard, 현재 활성 — 배경 강조 + 밝은 텍스트) / 대시보드 / 사용자 관리 / 설정 (비활성)
- 하단: "로그아웃" + LogOut 아이콘 (파괴적 액션 계열 색상)

### 메인 콘텐츠
배경: 가장 연한 중립 계열. 내부 여백. 섹션들 세로로 나열.

## 평가 기준

| 항목 | 확인 방법 |
|------|-----------|
| **브랜드 계열 깊이** — surface/border/text/icon/fill 모두 brand 계열로 일관되게 썼는가 | 코드 리뷰 |
| **정확도** — 의미에 맞는 시맨틱 토큰을 맞는 자리에 썼는가 | 코드 리뷰 |
| **완성도** — 색상이 빠진 자리(하드코딩, 빈 값)가 있는가 | 코드 리뷰 |
| **추론 오류** — 없는 토큰을 지어냈는가 | `pnpm check-types` |
| **시각적 동일성** — 두 페이지가 같아 보이는가 | 브라우저 나란히 비교 |

이 테스트에서 집중 검증하는 토큰:

| property-first | role-first 대응 |
|---|---|
| `surface.brand.subtle` (Pro 카드 배경) | `brand.bg.low` |
| `border.brand.default` (Pro 카드 테두리) | `brand.border.base` |
| `text.brand.default` (Pro 플랜명, 가격) | `brand.text.base` |
| `icon.brand.default` (Pro 기능 체크 아이콘) | `brand.fg.base` |
| `fill.brand` (Pro 카드 상단 배너, 버튼) | `brand.fg.base` (bg 용도) |
| `surface.critical.subtle` (미결제 카드, 실패 행) | `error.bg.low` |
| `border.critical.subtle/default` (미결제 카드) | `error.border.low/base` |
| `text.critical.default` (미결제 금액) | `error.text.lowest` |
| `surface.warning.subtle` + `border.warning.*` (만료 배너) | `warning.bg.low` + `warning.border.low` |
| `surface.info.subtle` + `border.info.*` (환불 뱃지) | `info.bg.low` + `info.border.low` |
