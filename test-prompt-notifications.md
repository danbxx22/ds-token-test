# 토큰 ergonomics 테스트 프롬프트 — 알림 센터 페이지

아래 두 프롬프트를 **각각 별개의 새 대화**에서 실행한다.
프롬프트 내용은 import 경로와 출력 파일 경로만 다르고 UI 스펙은 동일하다.

이 테스트의 목적: 5가지 의미 역할(완료/오류/주의/안내/브랜드)을 **같은 카드 구조**에 반복 적용해 AI가 일관되게 시맨틱 토큰을 매핑하는지 검증.
추가 커버: 읽음/안읽음 상태, overlay.action, 상태별 요약 카운터 카드.

## 평가 기준

| 항목 | 확인 방법 |
|------|-----------|
| **일관성** — 같은 구조(카드)에 5가지 역할을 일관되게 적용했는가 | 코드 리뷰 |
| **읽음/안읽음 구분** — 배경 색상으로 상태를 올바르게 표현했는가 | 코드 리뷰 |
| **정확도** — 의미에 맞는 시맨틱 토큰을 맞는 자리에 썼는가 | 코드 리뷰 |
| **추론 오류** — 없는 토큰을 지어냈는가 | `pnpm check-types` |
| **시각적 동일성** — 두 페이지가 같아 보이는가 | 브라우저 나란히 비교 |

이 테스트에서 집중 검증하는 토큰:

| property-first | role-first 대응 |
|---|---|
| `surface.brand.subtle` (읽지 않음 배경) | `brand.bg.low` |
| 왼쪽 강조선 `fill.positive/critical/warning/info/brand` | `success/error/warning/info/brand.fg.base` |
| `icon.*.default` (각 역할) | `*.fg.base` |
| `surface.*.subtle` + `border.*.subtle` + `text.*.default` (뱃지) | `*.bg.low` + `*.border.low` + `*.text.lowest` |
| 읽음 상태 강조선 (subtle fill) | 같은 계열의 low 단계 |
